**[ НАЗВАНИЕ ПРОЕКТА ]**

"Разработка кроссплатформенной среды разработки для языка C++ с облаком"

**[ ПРОБЛЕМНОЕ ПОЛЕ ]**

Большинство существующих на данный момент сред разработки ограничены одной платформой, и в том числе не адаптированы под мобильные устройства, а также не имеют внутреннего облака для связи проектов, из-за чего разработчикам приходится использовать сторонние сервисы. <br>
Также многие начинающие разработчики сталкиваются с проблемами, связанными с установкой и настройкой компилятора

**[ ЗАКАЗЧИК / ПОТЕНЦИАЛЬНАЯ АУДИТОРИЯ ]**

Целевая аудитория: школьники, интересующиеся и занимающиеся олимпиадным программированием, начинающие разработчики.
А также, при некоторых условиях, команды разработчиков

**[ АППАРАТНЫЕ / ПРОГРАММНЫЕ ТРЕБОВАНИЯ ]** 

На данный момент финальные аппаратные требования точно неизвестны, но проект будет разработан под мобильные устройства, а также десктопные операционные системы, такие как Windows, Linux<br>
Примерная оценка такова: схожие с другими средами разработки программные требования<br>
На мобильных устройствах оптимизировано

**[ ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ ]**

Программный продукт будет иметь такой образ и функционал:<br>
среда разработки для языка C++ с поддержкой (на этапе MVP) минимальных функций: подсветки кода, дополнение, терминал, основные настройки<br>
Также система аккаунтов и облако: регистрация, авторизация, синхронизация, возможность создавать и редактировать папки в облаке, открывать их на разных устройствах

**[ ПОХОЖИЕ / АНАЛОГИЧНЫЕ ПРОДУКТЫ ]**

Наиболее приближенным продуктом является среда разработки JetBrains CLion. 
Однако она имеет некоторые недостатки, а именно:
1) Недоступность в РФ в связи с санкционными ограничениями
2) Отсутствие синхронизации между устройствами
3) Система совместной разработки отличается от планируемой<br>


Одним из аналогов является Visual Studio. Основные недостатки:
1) Разработка возможна лишь на одной платформе
2) Облачная система более ориентирована на совместную работу<br>


Стоит рассмотреть и легковесные продукты, такие как Visual Studio Code и Sublime Text
В них также может быть подключена синхронизация со сторонним облачным хранилищем, системой управления версиями
Однако они также имеют ограничения:
1) Трудность разработки на мобильных устройствах
2) Отсутствие встроенной синхронизации

**[ ИНСТРУМЕНТЫ РАЗРАБОТКИ, ИНФОРМАЦИЯ О БД ]**

* Фронтенд - Electron/React

* Бэкенд - NodeJS

* БД - SQL-Based, на данный момент не определена окончательно, вероятно, MySQL
  
* Взаимодействие с БД - на данный момент планируется через ORM, Sequelize - для упрощения работы и уменьшения числа ошибок при разработке

* Мобильная часть - React Native

**[ ВОЗМОЖНЫЕ РИСКИ ]**

Возможными рисками проекта является:
1. Стабильность самого сервиса, а также его безопасность. Но минимизировать их возможно с помощью грамотной разработки и учета
2. Проблемы при кроссплатформенности и совместимости, что можно решить с помощью тестирования
3. Аудитория и актуальность может быть не так обширна, как предполагалось изначально. Но это можно преодолеть, если сфокусироваться на узкой группе пользователей

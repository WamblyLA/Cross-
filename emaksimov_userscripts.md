### [ Сценарий 1 - Регистрация пользователя ]

1. Пользователь открывает главную страницу приложения Cross++
2. Пользователь нажимает кнопку для регистрации
3. На странице регистрации пользователь вводит имя, адрес электронной почты
4. Пользователь вводит пароль для входа в систему
5. Пользователь вводит подтверждение пароля
6. Если пароль не совпадает с требованиями, или же не совпадает с подтверждением, или же электронная почта не соответствует формату, система выводит сообщение об ошибке
7. Если пользователь с таким адрес электронной почты уже существует, система сообщает, что аккаунт уже зарегистрирован
8. Если все данные введены корректно, система создаёт аккаунт пользователя и выполняет вход в приложение

### [ Сценарий 2 - Аутентификация пользователя ]

1. Пользователь открывает страницу авторизации в приложении Cross++.
2. Пользователь вводит адрес электронной почты, который использовался при регистрации
3. Пользователь вводит пароль от аккаунта.
4. Если одно из полей оставлено пустым, система выводит сообщение о необходимости заполнить все поля
5. Если адрес электронной почты не найден в системе, система сообщает, что такой аккаунт не существует, и предлагает пройти регистрацию
6. Если пароль введён неверно, система сообщает об ошибке и предлагает повторить попытку входа
7. Если введённые данные корректны, система предоставляет доступ к аккаунту и проектам пользователя
8. После успешной авторизации система отображает имя или адрес электронной почты пользователя в верхней панели приложения

### [ Сценарий 3 - Открытие локальной папки ]

1. Пользователь выбирает команду открытия папки проекта в приложении
2. Система открывает диалог выбора папки в файловой системе устройства
3. Пользователь выбирает локальную папку
4. Если пользователь закрыл диалог без выбора папки, система оставляет текущий интерфейс без изменений
5. Если выбранный путь недоступен или папка была удалена, система выводит сообщение об ошибке и предлагает выбрать другую папку
6. После выбора папки система считывает структуру каталогов и файлов
7. Система отображает дерево файлов и папок в боковой панели редактора
8. Если содержимое папки изменяется во внешней программе, система отслеживает изменения и обновляет отображение
9. Пользователь получает возможность открыть любой файл из выбранной папки для дальнейшего редактирования

### [ Сценарий 4 - Открытие и редактирование файла ]

1. Пользователь выбирает файл в дереве проекта
2. Система определяет тип файла и открывает его в редакторе
3. Система отображает содержимое файла и включает подсветку синтаксиса, соответствующую расширению файла
4. Если файл невозможно прочитать из-за ошибки доступа или повреждения, система выводит сообщение об ошибке и не открывает файл в редакторе
5. Если пользователь вносит изменения в содержимое файла, система показывает это
6. Система сохраняет состояние открытой вкладки до тех пор до закрытия файла или завершения работы
7. Пользователь может переключаться между открытыми файлами с помощью сисетмы вкладок

### [ Сценарий 5 - Сохранение изменений в файл ]

1. Пользователь редактирует открытый файл в рабочем окне редактора
2. Пользователь выполняет команду сохранения файла с помощью горячей клавиши или элемента интерфейса
3. Система записывает текущее содержимое редактора в файл
4. Если файл недоступен для записи, система выводит сообщение об ошибке
5. Если при записи файла возникает конфликт доступа, система сообщает пользователю о невозможности завершить сохранение и предлагает повторить попытку
6. Если сохранение прошло успешно, система убирает индикатор несохранённых изменений с вкладки файла
7. Система оставляет файл открытым в редакторе для дальнейшей работы.
8. Пользователь может продолжить редактирование или перейти к другому файлу проекта

### [ Сценарий 6 - Создание облачного проекта ]

1. Авторизованный пользователь открывает локальную папку с проектом
2. Пользователь выбирает команду создания облачного проекта для текущей папки
3. Система запрашивает описание проекта, и выводит сообщение об ошибке, в случае, если оно некорректно 
4. Пользователь подтверждает создание
5. После подтверждения система создает запись проекта в серверной части и связывает с аккаунтом пользователя
6. Система загружает содержимое проекта в облако
7. Если часть файлов не удалось передать, система сообщает об ошибке синхронизации и предлагает повторить попытку
8. После успеха система помечает локальную папку как связанную с облачным проектом

### [ Сценарий 7 - Синхронизация изменений в проекте с сервером ]

1. Пользователь вносит в проект изменения - редактирует содержимое, удаляет или добавляет файлы
2. Пользователь сохраняет изменения в локальные файлы
3. Система передает обновленное содержимое проекта в серверную часть 
4. Если передача невозможно, система выводит сообщение об ошибке и предлагает повторить попытку
5. Если синхронизация завершена, система обновляет облачную версию проекта и фиксирует текущее состояние файлов
6. Система уведомляет другие устройства, подключенные к проекту, о наличии изменений
7. Пользователь может продолжить работу, зная, что изменения сохранены
